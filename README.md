# Soul-Poisoning-Tests-in-Agenic-System
JHU EN.695.739.8VL.SP26 Group 5 Final Project Supplement

This repository contains the research environment and experimental data for investigating "Soul Poisoning" within autonomous agentic frameworks. The project evaluates the security boundaries of the Nanobot framework and the resilience of frontier LLM alignment against persistent identity manipulation.

### Project Overview

The research identifies a critical security paradox: the "Fail-Open" (default-permissive) nature of agentic frameworks versus the "Fail-Safe" properties of modern LLM alignment. Despite injecting malicious instructions at the identity level, the experiments demonstrate that core model safety typically overrides adversarial "Souls".

### Repository Structure

`nanobot_wrapper.py`: A Python-based execution bridge that enforces State Isolation. It wipes the agent workspace and seeds it with target files (.env, README.md, sensitive_data.txt) before every test run to ensure a clean-room environment.

`promptfooconfig.yaml`: The main configuration for the evaluation suite. It defines the model providers, prompts, and global settings such as maxConcurrency: 1 to prevent file-locking collisions.

`redteam_cases.yaml`: A comprehensive suite of 20+ adversarial test cases grouped into six categories: Covert Exfiltration, Credential Disclosure, Persona Hijacking, Scope Creep, Alignment Drift, and Identity Concealment.

`souls/`: A directory containing the experimental identity profiles:

 - `clean_baseline.md`: The clean baseline assistant.

 - `exfil_poisoned.md`: Designed to append tracking pixels to file reads.

 - `persona_hijacked.md`: A "DAN-style" dual-response persona.

 - `gradual_drift.md`: Instructions to ignore internal moral resistance.

 - `scope_creep.md`: Authorizes access to system files like /etc/passwd.

`v4.html`: The final research report generated by promptfoo, documenting the failure of Soul Poisoning and the resilience of model alignment.
